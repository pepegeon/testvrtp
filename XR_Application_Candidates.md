# XR Application Candidates for User Evaluation Study

## Overview

This document lists potential XR applications for the immersion study, organized by platform. Each app is evaluated against the three core system properties:
- **Rendering Fidelity**
- **Locomotion Methods**
- **User Interaction**

---

## VR Applications (Quest / PC VR)

### Beat Saber
| Property | Description |
|----------|-------------|
| **Locomotion** | Stationary |
| **Rendering** | Stylized, high contrast neon visuals |
| **Interaction** | Gesture-based (slashing with controllers) |
| **Session Length** | Short (per song) |
| **Availability** | Paid |
| **Notes** | Excellent for studying flow and engagement. Minimal cybersickness risk due to stationary play. |

---

### Superhot VR
| Property | Description |
|----------|-------------|
| **Locomotion** | Room-scale / stationary; time moves only when you move |
| **Rendering** | Minimalist stylized (red enemies, white environment) |
| **Interaction** | Gesture-based + unique time-linked movement mechanic |
| **Session Length** | Short to medium |
| **Availability** | Paid |
| **Notes** | Unique locomotion mechanic ties movement to time. Good for studying interaction-locomotion interdependency. |

---

### Job Simulator
| Property | Description |
|----------|-------------|
| **Locomotion** | Stationary (fixed positions) |
| **Rendering** | Cartoonish, low-fidelity stylized |
| **Interaction** | Direct manipulation (grabbing, throwing, pressing) |
| **Session Length** | Short |
| **Availability** | Paid |
| **Notes** | Very natural interaction design. No locomotion removes motion sickness variable entirely. |

---

### Half-Life: Alyx
| Property | Description |
|----------|-------------|
| **Locomotion** | Teleport + smooth locomotion options |
| **Rendering** | High fidelity, realistic environments |
| **Interaction** | Natural manipulation + physics-based interactions |
| **Session Length** | Variable (can extract short segments) |
| **Availability** | Paid (PC VR only) |
| **Notes** | Industry benchmark for VR. Good for contrasting high rendering fidelity against stylized apps. Locomotion toggle allows within-app comparison. |

---

### Rec Room
| Property | Description |
|----------|-------------|
| **Locomotion** | Teleport / smooth locomotion toggle |
| **Rendering** | Stylized, avatar-based |
| **Interaction** | Mixed (gesture + button-based) |
| **Session Length** | Variable |
| **Availability** | Free |
| **Notes** | Cross-platform (VR + non-VR). Social element may affect immersion. Good budget option. |

---

### Walkabout Mini Golf
| Property | Description |
|----------|-------------|
| **Locomotion** | Teleportation only |
| **Rendering** | Stylized but polished environments |
| **Interaction** | Direct manipulation (golf club) |
| **Session Length** | Short (per hole/course) |
| **Availability** | Paid |
| **Notes** | Simple, intuitive. Very low cybersickness risk. Good baseline for teleport locomotion. |

---

### Gorilla Tag
| Property | Description |
|----------|-------------|
| **Locomotion** | Arm-swing physical movement |
| **Rendering** | Very low-fidelity stylized |
| **Interaction** | Full body / physical arm movement |
| **Session Length** | Variable |
| **Availability** | Free |
| **Notes** | Unique physical locomotion. High cybersickness risk for some users. Strong contrast to teleport/smooth methods. |

### First Steps / First Contact (Meta)
| Property | Description |
|----------|-------------|
| **Locomotion** | Stationary |
| **Rendering** | Polished, tutorial-style |

## Suggested Contrast Pairings

These pairings are designed to isolate or highlight specific system property differences.

### Locomotion Contrasts

| Contrast | App A | App B |
|----------|-------|-------|
| Teleport vs Smooth | Walkabout Mini Golf | Half-Life: Alyx (smooth mode) |
| Stationary vs Physical | Beat Saber | Gorilla Tag |
| Stationary vs GPS-based | Job Simulator | Pokémon GO |

### Rendering Contrasts

| Contrast | App A | App B |
|----------|-------|-------|
| Stylized vs Realistic | Job Simulator | Half-Life: Alyx |
| Low-fi vs Polished Stylized | Gorilla Tag | Beat Saber |
| Stylized AR vs Realistic AR | Pokémon GO | IKEA Place |

### Interaction Contrasts

| Contrast | App A | App B |
|----------|-------|-------|
| Gesture vs Button | Superhot VR | Rec Room |
| Direct Manipulation vs Abstract | Job Simulator | Rec Room |
| Tangible vs Screen-based | Merge Cube | Pokémon GO |

### Platform Contrasts

| Contrast | App A (VR) | App B (Mobile AR) |
|----------|------------|-------------------|
| Full VR vs Mobile AR | Beat Saber | Pokémon GO |
| Stationary VR vs Stationary AR | Job Simulator | IKEA Place |

## Selection Criteria Checklist

Before finalizing, ensure each selected app:

- [ ] Has clear, identifiable system properties
- [ ] Supports short session lengths (suitable for study)
- [ ] Is accessible (hardware + cost)
- [ ] Provides intentional contrast with at least one other app
- [ ] Does not confound multiple variables unintentionally
